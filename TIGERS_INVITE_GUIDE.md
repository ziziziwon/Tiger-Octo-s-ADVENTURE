# 🎟 타이거즈의 초대장 (Tigers' Secret Invite)

## 📖 개요
"타이거즈의 초대장"은 모든 추억을 수집한 플레이어만이 경험할 수 있는 특별한 **After Credit Scene**입니다.

단순한 엔딩이 아닌, **"관객이 세계의 일부가 되는 순간"**을 연출하는 포트폴리오 안의 또 하나의 공연입니다.

- **모든 아이템 수집** (10개+)
- **모든 메모리 카드 수집** (6개+)
- 조건 충족 시 **갤러리 페이지**에 반짝이는 초대장 카드 자동 등장

### 2️⃣ 초대장 카드 (InviteCard)
- 💎 **디자인**: 황금빛 그라데이션 (#fffbea), 반짝이는 애니메이션, 홀로그램 효과

## 🛠️ 기술 구현

### 컴포넌트 구조
```
src/
├── components/
│   ├── InviteCard.tsx          # 갤러리의 초대장 카드
│   ├── InviteScene.tsx         # 초대장 페이지 (Canvas 별 파티클)
│   └── MemoryStage.tsx         # 기억의 무대 (보상 씬)
├── utils/
│   └── posterGenerator.ts      # 포스터 동적 생성 & 다운로드 로직
└── App.tsx                     # 라우팅 (/invite, /memory-stage)
```

### 주요 기술

#### 1. Glitch 전환 효과 (InviteCard)
```typescript
// Phase 1: Glitch 시작
body.style.filter = 'hue-rotate(30deg) contrast(1.3)';
body.style.mixBlendMode = 'difference';

// Phase 2: Flash (100ms 후)
body.style.filter = 'brightness(3)';

// Phase 3: 정상화 & 페이지 전환 (300ms 후)
body.style.filter = '';
body.style.mixBlendMode = '';
navigate('/invite');
```

#### 2. Canvas Particle System (InviteScene)
- `requestAnimationFrame` 기반 별 애니메이션
- 150개의 별, 각각 독립적인 반짝임 phase (sin 함수)
- 큰 별(size > 1.5)에 십자 반짝임 추가
- 느린 하강 이동 (speedY: 0.1-0.4)
- 화면 밖으로 나가면 상단에서 재생성
- 파스텔 색상 배열 랜덤 적용

#### 3. 보상 모달 애니메이션 (MemoryStage)
- Framer Motion `initial/animate` 패턴
- 순차 등장 (delay: 0.2s → 0.4s → 0.5s → 0.7s)
- 버튼 hover/tap 효과 (scale, shadow)
- 배경 스포트라이트 pulse 애니메이션

#### 4. 동적 포스터 생성 (posterGenerator.ts)
- **Canvas API 활용**:
  - `createLinearGradient()`: 배경 그라데이션
  - `arc()`: 별, 극단이 실루엣, 장식 링
  - `setLineDash()`: 점선 효과
  - `fillText()`: 텍스트 렌더링 (이모지 포함)
  - `shadowBlur`: 글로우 효과
- **Data URL 변환**: `canvas.toDataURL('image/png')`
- **자동 다운로드**: `<a>` 태그 생성 → click → 제거

### 성능 최적화

## 📝 개발 노트

### 개발 기간
- **기획**: 1시간
- **구현**: 4시간 (Canvas 포스터 생성 포함)
- **테스트 & 최적화**: 1.5시간
- **총 개발 시간**: ~6.5시간

### 주요 챌린지
1. **Canvas 파티클 성능**: 
   - 문제: 별 200개 이상에서 프레임 드롭
   - 해결: 150개로 제한, 반짝임 계산 최적화
2. **Glitch 효과 타이밍**: 
   - 문제: 너무 길면 어지럽고, 짧으면 인지 안 됨
   - 해결: 300ms 타이밍, brightness flash로 강조
3. **포스터 폰트 렌더링**: 
   - 문제: Canvas에서 커스텀 폰트 로딩 불안정
   - 해결: 시스템 폰트 + 이모지 조합으로 대체
4. **크로스 브라우저 호환성**: 
   - 문제: Safari에서 mix-blend-mode 이슈
   - 해결: 짧은 타이밍으로 부작용 최소화

### 배운 점
- Canvas API는 복잡한 그래픽에 강력하지만, 폰트 로딩은 주의 필요
- Framer Motion의 `transition.delay`로 감정 곡선 제어 가능
- requestAnimationFrame은 반드시 cleanup 필요 (메모리 누수 방지)
- 사용자 경험은 타이밍이 핵심 (300ms의 차이가 몰입도를 좌우)
